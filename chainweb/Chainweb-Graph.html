data ChainGraphException :: Type where Source #

This exceptions are not about the properties of the graph itself but about properties of enties (BlockHeader graph) that are constrained by this graph. So, maybe we should move this and the respective checks to the place where those enties are defined and rename these exceptions accordingly. However, keeping it here remove code duplication.

toChainGraph :: (a -> ChainId) -> KnownGraph -> DiGraph a -> ChainGraph Source #

This function is unsafe, it throws an error if the graph isn't a valid chain graph. That's OK, since chaingraphs are hard-coded in the code and won't change dynamically, except for during testing.

validChainGraph :: DiGraph ChainId -> Bool Source #

A valid chain graph is symmetric, regular, and the out-degree is at least 1 if the graph has at least two vertices.

These properties imply that the graph is strongly connected.
