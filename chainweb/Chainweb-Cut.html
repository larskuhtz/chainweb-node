data Cut Source #

A cut is a distributed state of a concurrent computation. Formally, a cut satisfies the property

  • prop_cutBraiding

A cut also satisfies the properties that

  • the graph is valid,
  • the graph corresponds to the ChainwebVersion,
  • the set of ChainIds of the cut are exactly the vertices of the graph,
  • all blockHeaders are valid (which is guaranteed by genesisBlockHeader) with respect to the graph.

limitCut :: HasCallStack => WebBlockHeaderDb -> BlockHeight -> Cut -> IO Cut Source #

Find a Cut that is a predecessor of the given one, and that has a block height that is smaller or equal the given height.

If the requested limit is larger or equal to the current height, the given cut is returned.

Otherwise, the requested height limit is divided by the number of chains (rounded down) and the result is used such that for each chain the predecessor of the given cut at the respective height is returned.

Check that a cut is correctly braided.

Extending Cuts

isMonotonicCutExtension :: MonadThrow m => Cut -> BlockHeader -> m Bool Source #

Extends a Cut monotonically, i.e. the replaced block header is the parent of the added block header.

Checks

  • block header is from the ChainGraph of the Cut
  • result has valid braiding
  • result is a cut
  • update is monotonic

This includes a check that inductively maintains checkBraidingOfCut.

joinIntoHeavier :: WebBlockHeaderDb -> Cut -> Cut -> IO Cut Source #

Merge two Cuts. If at least one of the input cuts had a valid braiding the result is guaranteed to have a valid braiding for all blocks included in cut and their ancestors.

This is because the merge starts with the intersection of both cuts, using branchDiff_ on each chain, and constructs the merge cut using tryMonotonicCutExtension. If one of the inputs is correctly braided, so is the intersection. tryMonotonicCutExtension is guaranteed to maintain that property.
