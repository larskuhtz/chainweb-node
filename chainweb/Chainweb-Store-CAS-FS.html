data FsDB t Source #

A content-addressed store for block payloads using a direct filesystem representation. Block payload hashes are converted to hex strings and mapped to files rooted at the given directory. We use a 3-deep nesting, e.g. block payload hash

 f22136124cd3e1d65a48487cecf310771b2fd1e83dc032e3d19724160ac0ff71

would be stored at

 $ROOT/f22/136/124cd3e1d65a48487cecf310771b2fd1e83dc032e3d19724160ac0ff71

Filesystem operations are abstracted by passing an FsOps record; this will allow us to plug in a backend using S3 later.
